# Weapon 3D Models

Low-poly stylized weapon meshes for the MCP Souls game, generated with bpy
(Blender Python API) and imported as .glb files.

## Regenerating models

The models are generated by `tools/gen_weapons.py` using `bpy` (headless
Blender). To regenerate:

```bash
"/mnt/c/Program Files/Blender Foundation/Blender 4.0/blender.exe" \
    --background --python tools/gen_weapons.py
```

This produces 6 .glb files (one per weapon) in this directory. Godot
reimports them automatically on next startup, or run:

```bash
~/.local/bin/godot --headless --import
```

## Data-driven attachment

The mapping `weapon_id → .glb` is **NOT** hardcoded in `player.gd`. Each
`data/weapons/<id>.tres` declares its own:

- `model_path` — `res://` path to the .glb (e.g. `res://assets/models/weapons/wpn_short_sword.glb`)
- `model_rotation` — extra Euler XYZ rotation in degrees (default 0,0,0)
- `grip_offset` — distance from model origin to grip point (meters)
- `model_scale` — uniform scale multiplier (default 1.0)

This means the AI (via MCP) can add new weapons by creating a new
`data/weapons/<new_id>.tres` and a new `.glb` — no code change needed.

`Player._load_weapon_model()` reads the .tres and applies the correct
position/rotation/scale at runtime.

## Coordinate convention

Models exported from Blender with the default Z-up convention. The base
rotation in `Player._load_weapon_model` is -90° around X, which converts:

- +Z (handle up)     → +Y (up in Godot)
- +Y (blade forward) → -Z (forward in Godot)
- +X (right)         → +X (unchanged)

If you model a new weapon in Blender, follow the same convention:
- Handle along +Z (pommel at Z=0, tip at Z=positive)
- Blade forward along +Y
- Right side along +X
